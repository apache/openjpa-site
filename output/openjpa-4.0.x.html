OpenJPA 4.0.x¶

The Apache OpenJPA community is currently mainly working on Apache OpenJPA 4.0.x.

This distribution is based on the Jakarta Persistence API 3.0.

Additional information on the OpenJPA project may be found at the project web site.

Changes in OpenJPA 4.0.1¶

Bug ¶

  • [OPENJPA-2921] - upgrade bundle and karaf plugins to fix java17 build
  • [OPENJPA-2923] - PCEnhancer creates deprecated call to new Long(long)

Improvement ¶

  • [OPENJPA-2922] - disable JNDI lookup in OpenJPAPersistence by default

Task ¶

  • [OPENJPA-2924] - BlacklistClassResolver can be improved
  • [OPENJPA-2926] - Dependencies and maven plugins need to be updated

Changes in OpenJPA 4.0.0¶

Bug ¶

  • [OPENJPA-2915] - commons-dbcp2 2.10.0 breaks OpenJPA because of changed configuration methods

Improvement ¶

  • [OPENJPA-2908] - implement Jakarta JPA 3.1
  • [OPENJPA-2909] - implement field proxies with ASM instead Serp
  • [OPENJPA-2910] - implement WASManagedRuntime witout Serp
  • [OPENJPA-2911] - Replace Serp with native ASM code
  • [OPENJPA-2914] - upgrade xbean to 4.23
  • [OPENJPA-2917] - Subclass enhancement might fail on long fields
