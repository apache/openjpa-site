20.1.  Known issues with MariaDB

  • As of MariaDB 10.2 the DATETIME data type supports sub-second fractions. The default of MariaDB internally is to use no fractions. The number of fractions can be explicitly set via scale: @Column(scale=6) will lead to a DATETIME(6) column and able to store microseconds. A value of @Column(scale=-1) will explicitly turn off all fractions.
